# Role Onboarding & RACI Template

Use this template when introducing or documenting a role for a project.

## Role Onboarding Template
- Role name:
- Short role summary:
- Owner (person):
- Typical responsibilities:
- Primary contacts:
- Slack channels / mailing lists:
- Key documentation & links:
- Onboarding checklist:
  - [ ] Read project one-pager
  - [ ] Review backlog and acceptance criteria
  - [ ] Attend kickoff and planning
  - [ ] Confirm tooling access (CI/CD, dashboards)
  - [ ] Identify knowledge transfer needs

## RACI Table (example)
| Activity | Responsible | Accountable | Consulted | Informed |
|---|---:|---:|---|---|
| Define product requirements | Product Manager | Product Manager | BA, Design Lead | Developers, PM |
| Approve design | Design Lead | Product Manager | Developers | Stakeholders |
| Implement feature | Developers | Tech Lead | QA | PM |
| Test & approve release | QA Lead | Product Manager | DevOps, Developers | Stakeholders |
| Deploy to production | DevOps | DevOps | Developers, QA | PM, Stakeholders |

Notes:
- Responsible: does the work
- Accountable: signs off
- Consulted: offers input (two-way)
- Informed: kept updated (one-way)
